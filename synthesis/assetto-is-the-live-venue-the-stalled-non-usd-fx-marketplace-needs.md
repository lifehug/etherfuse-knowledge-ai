# Assetto is the already-Won venue the stalled non-USD FX marketplace cluster has been missing

> The Agora/Noble/Plume 'non-USD FX marketplace' has been stuck in Nurture for lack of an execution venue, but Assetto — same owner (Andres Salcedo), Won stage, already building a MXN/GBP/EUR/BRL/KRW liquidity hub on Stablebonds — is that venue, and Agora's stalled AUSD is its missing USD leg.

Assetto is building a **non-USD stablecoin liquidity hub backing wrapped MXN, GBP, EUR, BRL, and KRW tokens with Etherfuse Stablebonds** [wiki/entities/companies/assetto-precept-labs.md] — structurally the same product as the Agora/Noble/Plume 'non-USD FX marketplace' cluster already flagged in this graph. Two under-connected facts tie them together:

1. Assetto's defining Feb 23 discovery call was run by **Andres Salcedo** [wiki/entities/companies/assetto-precept-labs.md] — the same owner of the Agora/Noble/Plume cluster.
2. Assetto is **Won / KYB Engaged / onboarding open**, whereas the Agora/Noble/Plume leads sit in Nurture awaiting revival. Assetto is therefore the furthest-along instance of the same thesis and the only one with a live integration path.

**Non-obvious implication:** The stalled FX-marketplace package doesn't need to be *built* — it needs a USD leg routed into Assetto's already-committed non-USD hub. Assetto backs five non-USD currencies against Stablebonds but has no stated USD stablecoin for its FX pairs; Agora's AUSD (currently 'Nurture / low priority / awaiting response') is exactly that leg. Repackaging Agora's AUSD as the USD anchor *inside* Assetto's Won liquidity hub reuses a closed deal to revive a stalled one, rather than working four separate Salcedo-owned leads in parallel.

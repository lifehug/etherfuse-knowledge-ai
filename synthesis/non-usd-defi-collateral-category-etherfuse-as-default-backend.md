# Five inbound teams are building the same non-USD DeFi primitive on Stablebonds — productize the listing kit

> P0, Assetto, Rheo, Noble, and Stable independently arrived at building non-USD stablecoin/FX liquidity-and-lending infrastructure with Etherfuse Stablebonds as the common collateral backend — a new product category is forming where Etherfuse is the default primitive, and a standardized 'collateral-listing kit' would convert this inbound wave far faster than bespoke deals.

Across the neighborhood, the same thesis recurs from five different counterparties who reached Etherfuse separately, mostly inbound:

- **P0** wants all 6 Stablebonds listed as yield-bearing cross-venue margin collateral and an MXN FX rail, explicitly because 'USD-only DeFi [is] limiting for non-US users' [teams/sales/2026-04-21-0xyz-p0-won.md].
- **Assetto** is pivoting its whole company to 'a non-USD stablecoin liquidity hub on Solana — swaps + yield backed by Stablebonds,' citing competitors starting to build the same thing [teams/sales/2026-04-21-assetto-precept-labs.md].
- **Rheo** wants 'EM fixed income exposure' via CETES/TESOURO/KTB and rate-arbitrage products on Stablebond yields [teams/cfo/2026-04-21-rheo-partnership-structure.md].
- **Noble** is launching an 'FX DEX for non-USD stablecoins' and 'multi-currency yield stables' as the first app on its new chain [teams/ceo/docs/inbox/linear/2026-04-16-sales-178-noble.md].
- **Stable** wants an MXN yield vault distributed via its fintech network [teams/sales/2026-04-17-stable-chain.md].

They all converge on the same integration shape (LayerZero OFT bridging, Stablebonds as wrapped-token treasury, identical fee tiers of 25bps <4.5% yield up to 1.5% >10% appearing verbatim in Assetto and Rheo) [teams/sales/2026-04-16-assetto-precept-labs.md][teams/cfo/2026-04-21-rheo-partnership-structure.md]. This is no longer a series of one-off deals — it is a forming category in which Etherfuse is the de facto collateral layer. The leverage move: package a repeatable 'collateral-listing kit' (sandbox API keys, OFT bridge guide, oracle/price-gap handling, fee schedule, liquidity buffer spec) so each new inbound converts in days instead of months, and so Etherfuse owns the standard before a competitor commoditizes it — which Assetto warns is already starting.
